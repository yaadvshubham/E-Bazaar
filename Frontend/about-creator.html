Engineering Journal & Solved Challenges

Asynchronous Seeding & PostgreSQL Bloat Control
Resolved
Hurdle: When populating catalogs via seeder scripts or bulk scraper feeds, basic bulk insertion scripts crashed due to Supabase timeouts or database transaction memory bounds, and cleared user order ledgers unnecessarily.
Solution: Built an additive seeder wrapper that maps brand directories recursively. It implements batch insert buffers (100 rows per batch) and performs pre-insertion select queries to skip duplicate titles, protecting active order/user records.
Optimistic UI Caching & Timing Overlays
Resolved
Hurdle: Frequent network queries to Supabase resulted in brief flashes of raw pages, lowering E-Bazaar's premium aesthetic. But showing spinners on instant category shifts blocked the viewport and degraded UX.
Solution: Coded a client-side Stale-While-Revalidate (SWR) cache in `api.js` utilizing `localStorage`. Added a 300ms conditional delay threshold on loader overlays. Navigational or cached clicks render instantly with zero loader displays, while slow fetches (>300ms) fade in the brand overlay smoothly.
No-Transitions Theme & State Synchronization
Resolved
Hurdle: Navigating between catalog pages in Dark Mode caused bright-white blinks during initial rendering (before script loads), and elements re-animated their theme layout shift.
Solution: Injected self-executing inline theme-application scripts inside the `<head>` of all pages to set themes prior to body parsing. Created a `.no-transitions` style block to disable animations during page load, which is removed via a 120ms timeout on script readiness.

REST API Engine & Specifications

The Node.js and Express.js REST API exposes secure endpoints for catalog querying, authentication, wallet management, and order placement.

GET /api/products Fetch and search products with brand/category filters & SQLite/Postgres compatibility.
GET /api/products/:id Fetch details, description, price, and specs for a specific product ID.
POST /api/auth/register User signup, bcrypt encryption, JWT signing, & 150 INR welcome wallet bonus.
POST /api/auth/login User login, credentials verification, & JWT session token generation.
GET /api/auth/profile Retrieve user dashboard particulars (addresses, cart, wallet balances) via JWT.
POST /api/orders Place an order, deduct wallet balance, and log passbook transactions.
POST /api/orders/razorpay/create-order Create payment order IDs through direct Razorpay API integrations.
POST /api/orders/razorpay/verify-payment Verify checkout signatures securely and confirm ledger orders.

Deployment Architecture

E-Bazaar is architected with a decoupled hosting strategy, isolating the static client application from the dynamic REST API backend to ensure absolute scalability, failover redundancy, and sub-second execution speed.

Frontend Client

Hosted on Vercel
  • Edge Cache CDN: Frontend static layout served via globally distributed edge locations.
  • Continuous Integration: Auto-deployed from Git branch pushes with preview builds.
  • Optimization: Automatic assets compression, clean routing, and static page optimizations.

Backend REST API

Hosted on Render
  • Web Services Engine: Node.js backend containerized and auto-deployed.
  • DB Pooling: High-performance connection pool handling to Supabase database instances.
  • Health Monitoring: Auto-restarts, environment controls, and integrated log streams.
